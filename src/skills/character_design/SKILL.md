# 如何设计人物

## 适用场景

- 你有一个故事点子，但不知道主角该是什么样的人
- 你的人物被批评"扁平""工具人""没有代入感"
- 你想让人物拥有真实的欲望、缺陷和成长轨迹
- 你需要系统的方法而非零碎灵感

## 核心设计理念

一个让人记住的人物 = **欲望** + **缺陷** + **弧光**

## 第一步：人物速写（让角色"活"在纸上）

为人物建立基础档案，只选最关键的 5-10 条，其余作为幕后知识。

### 必填项

- **姓名、年龄、外貌标志性细节**（一道疤、一种习惯动作、总穿同一件外套）
- **职业、社会身份、经济状况**
- **核心性格**（用 2-3 个形容词，如：固执、幽默、敏感）
- **对待不同地位者的态度**（比自己高/相等/低的人）
- **压力下的第一反应**（战斗、逃跑、僵住、讨好？）
- **一个独特癖好或习惯**（紧张时转笔、睡前检查门锁三次）

### 检查点

- 这些特征中是否有至少一个会直接影响情节？
- 你是否知道他的习惯动作，而不仅是一堆形容词？

## 第二步：内在驱动（让人物"不得不"行动）

这是人物设计的核心，没有驱动，人物就是情节的工具人。

### 三大核心元素

**1. 外在目标（What he wants）**

- 故事里表面追求的东西，可衡量、可判断成败。
- 例：找到宝藏、赢得比赛、复仇、追到喜欢的人。

**2. 内在需求（What he needs）**

- 内心真正缺失的东西，往往自己没意识到，与成长有关。
- 例：需要学会信任、需要放下愧疚、接纳不完美的自己。

**3. 核心谎言（The Lie）**

- 因过往创伤而相信的错误信念。故事的任务就是击碎它。
- 例："我必须成为屠龙勇士才值得被认可""一旦示弱就会被伤害""有钱才能被爱"。

### 三者关系

- 谎言 = 内在需求的反面
- 外在目标 = 为了证明谎言而追求的虚假解药
- 内在需求 = 获得真正的幸福必须拥抱的真相

## 第三步：人物弧光（让角色在故事中"改变"）

### 四种常见弧光

| 弧光类型 | 起点    | 终点    | 经典例子        |
| ---- | ----- | ----- | ----------- |
| 成长弧  | 困于谎言  | 拥抱真理  | 傲慢→谦卑，懦弱→勇敢 |
| 堕落弧  | 怀有美德  | 走向毁灭  | 正义→偏执，善良→冷酷 |
| 考验弧  | 已具备美德 | 证明并坚持 | 诚实→在压力下依然诚实 |
| 不变弧  | 固执于谎言 | 拒绝改变  | 《了不起的盖茨比》   |

### 设计弧光三步法

1. **起点状态**：故事开始时，他相信什么谎言？有什么缺陷？
2. **过程事件**：哪些情节会依次挑战这个谎言？（3-4 次，逐步升级）
3. **终点状态**：结束时他变成了什么样？领悟了什么？

### 重要提醒

弧光必须由 **情节** 推动，不能靠画外音或角色突然想通。要让读者看到，是某一件具体的事、某一个人的话，让他内心发生了松动。

## 实操工具：人物访谈问卷（10问）

请用角色的口吻，以第一人称回答以下问题。如果某个问题让你情绪波动，说明角色活了。

1. 你最后悔的一件事是什么？
2. 你最害怕的东西是什么？（不要答"蜘蛛"，要答"被抛弃""像父亲一样失败"）
3. 你的口头禅或标志性动作是什么？
4. 你房间里最不想让别人看到的一样东西是什么？
5. 如果明天是世界末日，你今天会去做什么？
6. 你相信"好人有好报"吗？为什么？
7. 你曾经最崇拜的人是谁？现在呢？
8. 你对自己最大的误解是什么？
9. 在什么情况下你会打破自己最重要的原则？
10. 你内心深处最渴望听到别人对你说的一句话是什么？

## 常见陷阱与解药

| 陷阱    | 表现            | 解药                                |
| ----- | ------------- | --------------------------------- |
| 完美主角  | 长得帅、能力强、道德无瑕  | 给一个真实的弱点（不是"太善良"，而是"控制欲强""害怕冲突"）  |
| 工具人配角 | 角色只为了推动情节而存在  | 问：如果镜头不拍他，他私下在做什么？他想要什么？          |
| 性格分裂  | 行为前后矛盾，没有内在逻辑 | 回到"核心谎言"和"内在需求"，让每个行动都可从二者推导      |
| 过度共情  | 太爱角色，舍不得让他受苦  | 记住：读者的眼泪来自角色的痛苦。你越虐他（合理范围内），读者越爱他 |

# 

## 输出建议

当用户使用此 Skill 时，你可以：

- 引导用户按照三步法逐步回答关键问题
- 根据用户已有的点子，反向推导出合适的人物设定
- 对用户已有的人物进行诊断，指出缺陷和修改方向
- 提供随机提示（如从问卷中随机选一个问题让用户回答）
